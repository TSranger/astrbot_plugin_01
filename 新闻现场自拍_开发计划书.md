# 新闻现场自拍 — 开发计划书

> 创建时间：2026-06-25
> 最后更新：2026-06-25
> 目标：让 Bot 定时搜索世界级大新闻，编写"在现场看热闹"的口语文案，并生成一张 Bot 角色在新闻现场的"自拍图"一起发到群里。

---

## 开发边界（硬约束）

1. **只开发 `astrbot_plugin_01/` 文件夹内的内容**，不修改 AstrBot 主项目和其他插件。
2. 新增文件全部放在 `astrbot_plugin_01/` 下（如 `news_selfie.py`）。
3. 外部依赖通过 pip 安装（`uv add`），不手动改 `pyproject.toml` 以外的项目级文件。

---

## 新闻范围约束

- **只涉及地方性/区域性事件**：某地发生了什么（地震、火灾、发布会、游行、大型活动等）。
- **不涉及人物新闻**：排除某明星/某名人/某人怎么样了、某网红争议、娱乐圈八卦、人事任免等。
- LLM 评估阶段必须把"人物类新闻"明确标记为非大事并过滤掉。

---

## Bot 角色外观素材约定

插件从当前启用的 skill 文件夹中读取 Bot 外观信息，优先级从高到低：

| 优先级 | 素材 | 说明 |
|---|---|---|
| 1 | `skills/{skill}/skill.gif` | 角色 GIF 动图（有则用，无则跳过） |
| 2 | `skills/{skill}/images/` 目录 | 角色参考图片（1~多张 `.png/.jpg`，有则用） |
| 3 | `skills/{skill}/SKILL.md` 中的名字 | 仅有名字，靠画图模型自身对角色名的认知生成（如"派蒙""钟离""胡桃"等游戏角色名） |

**素材存在性不确定**：每种素材都可能存在也可能不存在，代码必须对每种情况做兜底。若 1 和 2 都没有，仅用名字调用图片生成 API——很多游戏角色靠名字就足够让模型知道长什么样。

---

## 总览

```
定时触发 → 拉取新闻头条 → LLM评估是否"大事" → 拉取新闻详情+图片
                                              ↓
                         生成自拍图 ← 角色外观(来自skill) ← LLM写文案
                                              ↓
                                         发送到群
```

### Bot 外观来源约定

插件按以下优先级从当前 skill 文件夹（如 `skills/漫游时空.skill/`）读取 Bot 外观：

```
优先级1: skill.gif          → 角色动图，如果存在则作为参考图传给 API
优先级2: images/*.png/.jpg  → 角色静态参考图，如果存在则全部作为参考图传给 API
优先级3: SKILL.md 中的名字   → 仅有名字，靠画图模型自身对角色名的认知生成
```

**具体逻辑**：

```python
def resolve_bot_appearance(skill_dir: Path) -> BotAppearance:
    gif_path = skill_dir / "skill.gif"
    images_dir = skill_dir / "images"
    reference_images: list[Path] = []
    reference_gif: Path | None = None

    if gif_path.exists():
        reference_gif = gif_path

    if images_dir.is_dir():
        for ext in (".png", ".jpg", ".jpeg", ".webp"):
            reference_images.extend(sorted(images_dir.glob(f"*{ext}")))

    # 从 SKILL.md 读取名字
    skill_md = skill_dir / "SKILL.md"
    bot_name = ""
    if skill_md.exists():
        content = skill_md.read_text(encoding="utf-8")
        # 提取 "你是：xxx岁，男/女，..." 行中的人设定位
        # 或直接取 metadata 中的 name 字段
        ...

    text_appearance = bot_name  # 仅有名字时，靠模型认知生成

    return BotAppearance(
        reference_gif=reference_gif,
        reference_images=reference_images,
        text_description=text_appearance,
    )
```

**传给图片生成 API 时的策略**：

- 如果有 `reference_gif` 或 `reference_images`：调用支持参考图输入的 API（如 MIMO），把参考图作为角色锚点
- 如果仅有 `text_description`（名字）：调用 DALL-E 3 / 通义万相等文字生图 API，靠角色名让模型自行生成。很多知名游戏角色（如"派蒙""钟离""胡桃"）靠名字就足够
- 如果 API 同时支持参考图+文字描述：两者都传，文字描述补充场景信息
