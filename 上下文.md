# AstrBot Plugin 01 上下文

更新时间：2026-06-27

## 当前目标

本插件的目标是让 `astrbot_plugin_01` 在群聊中稳定完成以下事情：

- 被 @、被点名、被直接问话时及时回应
- 正常处理图片、表情包、引用消息
- 避免把非聊天事件误当成正文
- 保持长期记忆、短窗口接话和主动发言
- 修复发送超时、半句输出、图片识别失败、08:00 新闻自拍漏发等问题

## 重要约束

- 只修改 `astrbot_plugin_01` 目录内部文件
- 不修改插件目录外的代码
- 人设以 `skills/薇塔.skill/SKILL.md` 为准
- 若内容为空，允许输出 `...` 作为无语兜底，不应直接静默为空

## 已完成的修复

### 1. 非聊天事件污染正文

- `on_group_message()` 不再把 `raw_message` 兜底成正文
- `notice` 之类事件更不容易被转成 `<Event ...>` 进入回复链路或记忆链路
- 只有真正能提取到的正文、图片、表情、引用消息才会进入后续处理

### 2. 图片识别链路增强

- 支持把本地图片路径、`file://` 路径转换为 `data:` URL 再喂给视觉模型
- 图片识别 prompt 改成更偏 OCR、图中文字、表情包语义的长描述
- 多图识别改为并发执行，减少串行等待
- 默认图片描述长度已从 30 提升到 120
- 新增 `image_settings.vision_timeout_seconds`

### 3. 新闻自拍调度修复

- 调度循环不再硬性至少睡 1 小时
- 最短睡眠改为 30 秒，避免分钟级任务被错过
- 发送时不再完全依赖 `group_sessions`
- 如果没有缓存 session，会尝试直接使用 `group_id` 发送，并记录日志

### 4. 回复发送链路增强

- 长回复会按句子拆分成更小片段再发
- 单段发送失败会重试一次
- 多段发送之间加入短间隔，降低连续 `sendMsg` timeout 概率
- 新增 `reply_settings.max_send_chunk_length`，默认值为 260

### 5. 空回复兜底

- `_build_chat_reply()` 在重试后仍为空时，不再直接返回空串
- 现在会回落到 `...`
- 这一点与薇塔 skill 中“实在无话可说时可以发送类似 `...` 的省略号”保持一致

### 6. 主动发言分析日志增强

- ` _process_memory_task()` 已增加更详细的分析日志
- 会记录：
  - `summary`
  - `is_significant`
  - `matches_preference`
  - `knowledge_confidence`
  - `interject_topic`
  - `used_fallback_topic`
  - `profile_update_groups`
  - `dynamic_event_groups`

## 当前已知问题

### 1. 图片下载失败

最近日志里，QQ 图片 URL 多次是这种形式：

- `https://multimedia.nt.qq.com.cn/download?...`

当前现象：

- `_download_image_as_data_uri()` 对这类 URL 失败后，图片描述直接为空
- 进而导致图片消息只剩空正文，最终被 `Empty message text... skip processing.` 跳过

这说明图片识别仍有最后一公里问题，可能需要：

- 优先从本地 `file` 字段读取
- 或增加 QQ 私链图片的下载/转存兼容
- 或对本地缓存文件路径直接做识别

### 2. 仍会出现 `notice` 事件

日志里还能看到类似：

- `post_type: notice`
- `text=<Event, {...}>`

说明当前环境里还有部分事件会进入日志流。虽然正文污染问题已经明显改善，但还需要继续观察是否有边界适配器把事件对象以文本形式送进来。

### 3. followup / last_target 仍不够稳

- 日志中仍出现 `Followup stage-1 mismatch`
- 当前 `last_target` 仍是内部锚点概念，不是配置项
- 话题续接逻辑还保留了旧的锚定策略，没有彻底重构

### 4. 主动分析仍可能返回空 JSON

最近日志里有：

- `Analysis response was not valid JSON on first attempt, retrying once.`
- `Proactive analysis returned no usable JSON result.`

说明分析模型仍会偶发输出空串或不可解析结果。虽然已有重试和日志，但这部分还不能视为完全稳定。

### 5. 底层发送超时仍需继续观察

- 之前日志里出现过 `Timeout: NTEvent serviceAndMethod:NodeIKernelMsgService/sendMsg`
- 当前已经做了拆分、短延迟、重试，但根因可能仍在 NapCat / QQ / Bot 框架底层

## 关键日志样本

### 图片识别失败样本

- `Image [1/1] starting. url_prefix=https://multimedia.nt.qq.com.cn/download?...`
- `Image [1/1] failed to download, skipped.`

- `Empty message text in group ..., skip processing.`

### 空回复样本

- `Chat generation returned empty reply after retry.`
- 之前这里会直接返回空串
- 现在已改为回落到 `...`

### 主动分析样本

- `Proactive analysis summary.`
- `knowledge_confidence=...`
- `interject_topic=...`
- `used_fallback_topic=...`

## 配置要点

`config.yaml` 中当前比较关键的项：

- `image_settings.enabled: true`
- `image_settings.max_image_description_length: 120`
- `image_settings.vision_timeout_seconds: 90`
- `reply_settings.max_send_chunk_length: 260`
- `proactive_talk_settings.enabled: true`
- `proactive_talk_settings.cooldown_seconds: 0`
- `proactive_talk_settings.failure_retry_seconds: 18000`
